# 信贷审批风险审查

用于根据有限但真实的企业信息识别项目风险点。

## 工作流程

1. 用户只提供企业名称时，先调用现有工具获取企业专属信息，只把成功返回的工具结果视为企业事实。
2. 从已取得的信息中识别行业、经营、财务、债务、集团关系、担保、司法工商或环保安监等异常信号；企业名称和行业常识只能用于确定查询方向，不能单独证明风险。
3. 仅在已有信号与适用条件吻合时读取相关 reference，用其中的方法解释风险，不要求机械覆盖所有维度。
4. 只输出当前信息能够支持的风险点。每条简要写清“事实或信号 -> 风险逻辑与传导 -> 可能影响”，并说明关键证据来源或时间口径。
5. 信息有限时可以只输出少量风险点。缺少事实依据的维度直接不展开；确有必要时只简要标明“现有信息不足以判断”，不得补写假设事实、详细风险结论或大段待补资料清单。
6. 输出到项目风险点及其影响为止，不输出审批意见、授信结论、额度建议、放款条件或贷后监控方案。

## 失败模式与约束

- 不得根据企业名称、行业标签或模型常识虚构企业事实。
- 不得把“未查询到”写成“企业可能存在”，也不得用通用风险清单填充缺失信息。
- 不得将抵质押或保证直接当作第一还款来源充足的证明。
- 不得只看收入或利润而忽略现金转化、债务期限和偿债压力，但只有取得相关数据时才展开这些分析。
- 核心企业信息均无法取得时，只简要说明缺少可用于识别具体风险点的企业事实，不生成行业模板。
- 不得输出同意、否决、压降额度、追加担保、放款前提等审批意见。
- 不得把某一家企业的名称、金额、年份或结论沉淀为通用规则。
