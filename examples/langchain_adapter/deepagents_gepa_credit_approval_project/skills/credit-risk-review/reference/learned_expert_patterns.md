# 审批专家经验提醒

本文件只沉淀审批官风险评价中可跨企业复用、且模型不容易稳定想起的风险提示。它用于提醒分析方向，不替代企业事实、完整调查流程或最终风险判断。

## 写入原则

- 每条通常使用一个标题和一段短文，写清“适用条件或信号 -> 重点关注 -> 可能后果”。
- 只有非显然且会改变风险识别或解释时，才补充一条关键核验或排除条件。
- 企业名称和行业词只能提示检索方向，不能单独触发风险结论。
- 不记录单一企业的名称、日期、金额、事件和结论，不把示例行业写成封闭名单。
- 不编造统一阈值；没有政策或数据依据时，使用企业历史、同业、合同约定或压力情景比较。
- 合并经济机制相同的经验。一次 proposal 只新增或修订少量提醒，不写成行业知识大全。
- 不重复常规财务分析、取证步骤和标准调查流程；这些内容由 `SKILL.md`、其他 reference 和模型通用能力负责。
- 缺少对应数据工具时记录为 `TOOL_CAPABILITY_GAP`，不得把审批意见中的隐藏事实写进本文件。

## 推荐格式

### 风险提示名称

当出现哪些可观察条件时，重点关注哪些信号或关系；它们可能通过什么路径影响现金流、偿债能力、经营持续性或风险缓释。必要时补充一个关键核验或排除条件。

## 已萃取提醒

仅在运行轨迹已有相关证据、智能体跳过了可用工具，或多条独立样本共同支持同一机制时新增或完善提醒。
